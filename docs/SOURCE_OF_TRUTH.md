## Product Definition

CampaignOps is an AI-assisted event and campaign operations platform.

It is an internal operating system for planning, executing, tracking, and learning from events and campaigns across government events, CSR campaigns, community events, cafe events, outreach campaigns, and client-facing execution work.

The product should help a team enter a raw event or campaign requirement and generate an execution-ready workspace with tasks, budget, vendors, outreach, timelines, risks, and reports.

## Product Positioning

CampaignOps is an AI-assisted event and campaign operations system that converts event requirements into execution plans using event taxonomy, scale tiers, budget patterns, stakeholder context, and reusable organizational knowledge.

Do not describe it as only:

- AI dashboard
- Event management tool
- CRM
- Chatbot
- Task manager

## Core Philosophy

AI is not the product. AI is one intelligent layer inside the product.

The system must first understand:

- Real business workflow
- Event type
- Scale
- Budget
- Timeline
- Stakeholders
- Human responsibilities
- Required vendors
- Department or client context
- Execution constraints

Then AI should generate structured outputs.

AI outputs must always be editable and reviewable by humans. No AI output is final without human approval.

## Final Product Rule

Any new idea must pass this test:

Does this support creating, planning, executing, tracking, or learning from an event/campaign?

If yes, it can be considered only if it belongs to one of the locked modules:

- Dashboard
- Event core
- Tasks
- Leads/contacts
- Vendors
- Budget
- Outreach
- Content calendar
- Files
- AI planner
- Playbook/knowledge
- Reports/debrief

If no, it goes to backlog.
